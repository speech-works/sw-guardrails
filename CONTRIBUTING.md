# Contributing: help us fix mistakes

> [!IMPORTANT]
> **This skill shapes how we write about a sensitive, lived experience, so it
> will contain mistakes.** Blind spots, outdated claims, mis-citations, tone that
> lands wrong, or framing that is too US- or Western-centric for an India
> audience are all likely. **If anything here looks wrong, please raise an issue
> so we can correct it.** You do not need to be sure. A one-line "this seems off"
> is enough to start.
>
> Corrections are especially welcome from **people who stutter or stammer, SLPs,
> and families.** If a guideline contradicts your lived experience, you are right
> and the rule needs to change, not you. Lived experience outranks any rule in
> this repository.

## What is worth flagging

- A rule that is wrong, harmful, or shaming.
- A claim that is outdated or not supported by evidence.
- A broken link, a mis-quote, or a source we represented unfairly.
- Framing that is too US/Western-centric, or that does not fit India.
- A missing source, perspective, or audience we should include.
- A grep pattern that fires on innocent text (a false positive).
- Anything in [`NOT-ADOPTED.md`](./NOT-ADOPTED.md) you think we should reconsider.

## How to raise an issue

1. **Search first.** Check [open issues](https://github.com/speech-works/sw-guardrails/issues)
   so we avoid duplicates.
2. **Open a new issue:** [create one here](https://github.com/speech-works/sw-guardrails/issues/new).
   (If you pick the "Correction or mistake" template, these fields appear for
   you automatically.)
3. **Copy the template below** into the description and fill in what you can.
   Skip anything that does not apply.
4. **Submit.** A maintainer will read it, discuss it with you if needed, and if
   we agree, update `SKILL.md` / the docs and credit the change.

## Copy-paste issue template

```markdown
### What might be wrong
<Quote the rule, claim, link, or phrasing. Add the file and section if you can,
e.g. SKILL.md > A7, or README references item 5.>

### Why it might be a mistake
<What is inaccurate, harmful, outdated, biased, or mis-cited?>

### Suggested correction
<What should it say instead? A rough idea is fine.>

### Source or evidence (optional)
<A link or citation that supports the correction.>

### Your perspective (optional)
<e.g. I stammer / I am an SLP / parent / researcher. Share only if you want to.>

### Severity (optional)
<blocker (actively harmful) / should-fix / minor>
```

That is it. Do not worry about getting the format perfect. We would much rather
have a rough, honest flag than a polished one that never gets filed.
